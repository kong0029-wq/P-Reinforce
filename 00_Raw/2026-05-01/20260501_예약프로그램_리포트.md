## 1. 수업 등록 폼 (Business Registration Form) 고도화

### 🕒 시간 설정 및 자동 생성 로직
- **시작/종료 시간 기반**: 사업자가 요일별로 수업 시작 시간과 종료 시간을 직접 지정합니다 (예: 10:00 ~ 18:00).
- **자동 타임 슬롯 생성**: 선택된 수업 시간 단위(30분 또는 1시간)에 따라 지정된 시간 범위 내에서 가능한 모든 수업 타임이 박스 형태로 자동 생성됩니다.
- **슬롯 ON/OFF 토글**: 자동 생성된 시간 박스 중 운영하지 않는 시간(점심시간, 휴식시간 등)은 클릭 한 번으로 비활성화(OFF) 처리할 수 있습니다. 비활성화된 시간은 수업 생성에서 제외됩니다.
- **일괄 설정(동일 적용)**: 특정 요일에서 설정한 시간 범위 및 ON/OFF 구성을 다른 모든 선택된 요일에 한 번에 복사하는 기능을 추가했습니다.

### 👥 인원 및 레슨 관리
- **강사명 필드 삭제**: 모든 수업 관리 프로세스에서 강사명 항목을 제거하여 수업 이름(ClassName) 중심으로 직관성을 높였습니다.
- **정원 설정 최적화**: 레슨 종류 선택 방식에서 1~6명의 숫자를 직접 선택하는 방식으로 변경하여 정원 설정 속도를 개선했습니다.

## 2. 사용자(회원) 예약 경험 개선

### 📅 예약 페이지 (Reservation.tsx)
- **수업 이름 표시**: 시간 버튼 아래에 수업 이름(예: 요가 기초반, 기구 필라테스)을 표시하여 회원이 어떤 수업을 예약하는지 명확히 알 수 있게 했습니다.
- **상세 정보 노출**: 예약 확인 영역에 수업의 종료 시간과 총 소요 시간(30분/60분) 정보를 추가했습니다.

### 📝 내 예약 내역 (MyReservations.tsx)
- **정보 일관성**: 예약된 수업 리스트에서 강사명 대신 수업 이름을 노출하도록 변경했습니다.
- **시간 표시 강화**: 수업이 몇 분 동안 진행되는지(30 min / 60 min) 실시간 데이터를 기반으로 표시합니다.

## 3. 기술적 변경 사항 (Technical Details)

### 📂 데이터 구조 변경
- `classes` 컬렉션: `endTime` (종료 시간), `duration` (소요 시간) 필드가 추가되었습니다. `instructor` 필드는 더 이상 사용되지 않습니다.
- `reservations` 컬렉션: 예약 시 해당 수업의 `className`, `classEndTime`, `classDuration` 정보가 함께 저장되도록 트랜잭션 로직을 업데이트했습니다.

### 🛠 주요 로직 구현
- `generateSlots` 유틸리티: 시작/종료 시간과 Duration을 계산하여 누락 없이 타임 슬롯을 생성하는 알고리즘 적용.
- `daySchedules` 상태 관리: 요일별 독립적인 스케줄(시작, 종료, 제외 시간)을 관리하기 위한 복합 객체 구조 도입.
