# Notion 날짜/요일 수식 및 템플릿 설정 정리

## 1. 확인했던 내용 요약

이번 대화에서는 Notion 데이터베이스에서 날짜 속성을 기준으로 요일을 표시하는 방법과, 생성된 데이터베이스 템플릿을 수정하거나 삭제하려고 할 때 발생하는 문제를 확인했습니다.

주요 내용은 다음과 같습니다.

1. 날짜 속성에서 요일만 표시하는 수식
2. 날짜와 요일을 함께 표시하는 수식
3. 한글 요일로 변환하는 방법
4. 데이터베이스 템플릿 수정 방법
5. 기본 제공 템플릿인 `비어 있음` 삭제 가능 여부
6. 삭제가 안 되는 경우의 대체 해결 방안

---

## 2. Notion에서 요일을 표시하는 기본 원리

Notion 데이터베이스에서 날짜를 기준으로 요일을 표시하려면 **수식 속성**을 사용합니다.

예를 들어 데이터베이스에 `작성 날짜`라는 날짜 속성이 있다면, 이 날짜를 기준으로 요일을 계산할 수 있습니다.

기본 함수는 다음과 같습니다.

```notion
formatDate(prop("작성 날짜"), "dddd")
```

이 수식은 `작성 날짜`에 입력된 날짜의 요일을 영어로 표시합니다.

예시 결과는 다음과 같습니다.

```text
Friday
```

---

## 3. 날짜와 요일을 함께 표시하는 방법

날짜뿐만 아니라 요일까지 함께 보이게 하려면 `formatDate()` 함수 안의 날짜 형식을 조합하면 됩니다.

예시 수식은 다음과 같습니다.

```notion
formatDate(prop("작성 날짜"), "YYYY년 M월 D일 dddd")
```

예시 결과는 다음과 같습니다.

```text
2026년 5월 1일 Friday
```

또는 조금 더 간단하게 표시하려면 다음처럼 사용할 수 있습니다.

```notion
formatDate(prop("작성 날짜"), "YYYY.MM.DD ddd")
```

예시 결과는 다음과 같습니다.

```text
2026.05.01 Fri
```

---

## 4. 한글 요일로 표시하는 방법

Notion의 `formatDate()` 함수는 기본적으로 요일을 영어로 표시합니다.

따라서 `금요일`, `월요일`처럼 한글 요일을 표시하려면 영어 요일을 한글로 변환하는 수식을 사용해야 합니다.

### 4.1 요일만 한글로 표시하는 수식

```notion
replaceAll(
  replaceAll(
    replaceAll(
      replaceAll(
        replaceAll(
          replaceAll(
            replaceAll(formatDate(prop("작성 날짜"), "ddd"),
            "Mon", "월요일"),
          "Tue", "화요일"),
        "Wed", "수요일"),
      "Thu", "목요일"),
    "Fri", "금요일"),
  "Sat", "토요일"),
"Sun", "일요일")
```

예시 결과는 다음과 같습니다.

```text
금요일
```

### 4.2 날짜와 한글 요일을 함께 표시하는 수식

```notion
formatDate(prop("작성 날짜"), "YYYY년 M월 D일 ") +
replaceAll(
  replaceAll(
    replaceAll(
      replaceAll(
        replaceAll(
          replaceAll(
            replaceAll(formatDate(prop("작성 날짜"), "ddd"),
            "Mon", "월요일"),
          "Tue", "화요일"),
        "Wed", "수요일"),
      "Thu", "목요일"),
    "Fri", "금요일"),
  "Sat", "토요일"),
"Sun", "일요일")
```

예시 결과는 다음과 같습니다.

```text
2026년 5월 1일 금요일
```

## 5. 확인된 화면 상황

첨부된 화면에서는 Notion 데이터베이스 페이지 안에 다음 속성들이 보였습니다.

| 속성 | 상태 |
|---|---|
| 날짜 | 금요일 |
| 작성 날짜 | 2026년 5월 1일 |
| 태그 | 비어 있음 |
| 템플릿 항목 | 새 페이지, 비어 있음, 새 템플릿 |

여기서 `날짜` 속성은 실제 날짜 입력 속성이 아니라, `작성 날짜`를 기준으로 계산된 **수식 속성**으로 보입니다.

즉, 구조는 다음과 같이 이해할 수 있습니다.

```text
작성 날짜 = 사용자가 입력하는 날짜
날짜 = 작성 날짜를 기준으로 요일을 표시하는 수식 속성
```

## 6. 문제점 1: 요일은 보이지만 날짜와 함께 보이지 않음

### 문제 상황

현재 화면에서는 `날짜` 속성에 `금요일`만 표시되고 있습니다.

즉, 사용자가 원했던 형태가 다음과 같다면,

```text
2026년 5월 1일 금요일
```

현재는 다음처럼 요일만 표시되는 상태입니다.

```text
금요일
```

### 원인

수식이 요일만 반환하도록 작성되어 있기 때문입니다.

예를 들어 다음과 같은 수식이면 요일만 표시됩니다.

```notion
formatDate(prop("작성 날짜"), "dddd")
```

또는 한글 변환 수식도 요일만 반환하도록 되어 있으면 결과가 `금요일`만 나옵니다.

### 해결 방안

`작성 날짜` 값과 한글 요일 값을 하나의 문자열로 합쳐야 합니다.

추천 수식은 다음과 같습니다.

```notion
formatDate(prop("작성 날짜"), "YYYY년 M월 D일 ") +
replaceAll(
  replaceAll(
    replaceAll(
      replaceAll(
        replaceAll(
          replaceAll(
            replaceAll(formatDate(prop("작성 날짜"), "ddd"),
            "Mon", "월요일"),
          "Tue", "화요일"),
        "Wed", "수요일"),
      "Thu", "목요일"),
    "Fri", "금요일"),
  "Sat", "토요일"),
"Sun", "일요일")
```

적용 후 예상 결과는 다음과 같습니다.

```text
2026년 5월 1일 금요일
```

## 7. 문제점 2: 템플릿 수정 위치가 헷갈림

### 문제 상황

Notion 데이터베이스 페이지 하단에 다음 항목이 보였습니다.

```text
새 페이지
비어 있음
새 템플릿
```

사용자는 여기서 만들어진 템플릿을 수정하고 싶어 했습니다.

### 원인

Notion에서는 일반 페이지와 데이터베이스 템플릿이 비슷하게 보입니다.

그래서 현재 열려 있는 페이지를 수정하는 것인지, 앞으로 새로 만들어질 페이지에 적용되는 템플릿을 수정하는 것인지 헷갈릴 수 있습니다.

### 해결 방안

템플릿을 수정하려면 데이터베이스의 템플릿 목록에서 해당 템플릿을 열어야 합니다.

일반적인 방법은 다음과 같습니다.

1. 데이터베이스 상단 또는 하단의 `새로 만들기` 또는 `New` 버튼 옆 화살표를 클릭합니다.
2. 템플릿 목록을 엽니다.
3. 수정할 템플릿 오른쪽의 `...` 버튼을 클릭합니다.
4. `편집` 또는 `Edit`을 선택합니다.
5. 템플릿 안의 속성, 본문, 기본값 등을 수정합니다.

수정한 템플릿은 이후 새 페이지를 만들 때 적용됩니다.

---

## 8. 문제점 3: `비어 있음` 템플릿을 삭제하려고 함

### 문제 상황

사용자는 화면에 보이는 `비어 있음` 템플릿을 삭제하고 싶어 했습니다.

하지만 `비어 있음`은 일반적으로 사용자가 직접 만든 템플릿이 아니라, Notion에서 기본 제공하는 항목입니다.

### 핵심 문제

`비어 있음`은 Notion의 기본 템플릿 역할을 하기 때문에 완전히 삭제할 수 없는 경우가 많습니다.

즉, 사용자가 만든 템플릿처럼 `삭제` 버튼이 나오지 않거나, 삭제해도 다시 보이는 것처럼 느껴질 수 있습니다.

---

## 9. `비어 있음` 템플릿 삭제가 어려운 이유

`비어 있음`은 말 그대로 아무 템플릿도 적용하지 않고 새 페이지를 만드는 기본 선택지입니다.

Notion은 데이터베이스에서 항상 새 빈 페이지를 만들 수 있도록 이 항목을 제공합니다.

따라서 사용자가 직접 만든 템플릿과 달리 완전 삭제 대상이 아닐 수 있습니다.

정리하면 다음과 같습니다.

| 항목 | 삭제 가능 여부 |
|---|---|
| 사용자가 직접 만든 템플릿 | 삭제 가능 |
| Notion 기본 `비어 있음` 항목 | 삭제 불가 또는 제한적 |
| 새 템플릿 버튼 | 삭제 대상 아님 |

---

## 10. 해결 방안: `비어 있음`을 삭제하지 않고 안 쓰는 방식으로 운영하기

`비어 있음`을 완전히 삭제할 수 없다면, 가장 좋은 해결책은 직접 만든 템플릿을 기본값으로 설정하는 것입니다.

### 권장 방식

1. 원하는 구조로 새 템플릿을 만듭니다.
2. 해당 템플릿에 필요한 속성, 본문, 기본값을 넣습니다.
3. 만든 템플릿을 기본 템플릿으로 설정합니다.
4. 앞으로 새 페이지를 만들 때 해당 템플릿이 자동으로 적용되도록 합니다.
5. `비어 있음`은 무시하고 사용하지 않습니다.

## 11. 추천 템플릿 구조

요일 자동 표시를 사용하는 데이터베이스라면 다음 구조를 추천합니다.

| 속성명 | 속성 유형 | 설명 |
|---|---|---|
| 작성 날짜 | 날짜 | 사용자가 실제 날짜를 입력하는 속성 |
| 날짜 | 수식 | 작성 날짜를 기준으로 날짜와 요일을 표시 |
| 태그 | 선택 또는 다중 선택 | 일정, 업무, 개인 등 분류 |
| 메모 | 텍스트 | 추가 설명 입력 |

`날짜` 수식에는 다음 수식을 넣는 것을 추천합니다.

```notion
formatDate(prop("작성 날짜"), "YYYY년 M월 D일 ") +
replaceAll(
  replaceAll(
    replaceAll(
      replaceAll(
        replaceAll(
          replaceAll(
            replaceAll(formatDate(prop("작성 날짜"), "ddd"),
            "Mon", "월요일"),
          "Tue", "화요일"),
        "Wed", "수요일"),
      "Thu", "목요일"),
    "Fri", "금요일"),
  "Sat", "토요일"),
"Sun", "일요일")
```

## 12. 기본 템플릿 설정 방법

직접 만든 템플릿을 기본으로 사용하려면 다음 순서대로 진행합니다.

1. 데이터베이스에서 `새로 만들기` 버튼 옆의 화살표를 클릭합니다.
2. 직접 만든 템플릿을 찾습니다.
3. 템플릿 오른쪽의 `...` 버튼을 클릭합니다.
4. `기본값으로 설정` 또는 `Set as default`를 선택합니다.
5. 이후 새 페이지를 만들면 해당 템플릿이 자동 적용됩니다.

이렇게 설정하면 `비어 있음`을 삭제하지 않아도 실질적으로 사용하지 않게 됩니다.

## 13. 실제 운영 시 추천 방식

가장 깔끔한 운영 방식은 다음과 같습니다.

```text
1. 작성 날짜는 날짜 속성으로 유지한다.
2. 날짜 또는 요일 표시용 속성은 수식 속성으로 만든다.
3. 한글 요일 변환 수식을 적용한다.
4. 직접 만든 템플릿을 기본값으로 설정한다.
5. 기본 제공 `비어 있음`은 삭제하지 않고 무시한다.
```

이 방식의 장점은 다음과 같습니다.

- 날짜를 한 번만 입력하면 요일이 자동 계산됩니다.
- 잘못된 요일을 직접 입력할 위험이 줄어듭니다.
- 새 페이지마다 동일한 구조가 자동 적용됩니다.
- 기본 템플릿 삭제 문제로 시간을 낭비하지 않아도 됩니다.

## 14. 최종 정리

이번에 확인한 핵심 내용은 다음과 같습니다.

### 확인 내용

- Notion에서는 `formatDate()` 함수로 날짜에서 요일을 추출할 수 있습니다.
- 기본 요일 출력은 영어입니다.
- 한글 요일은 `replaceAll()`을 이용해 변환해야 합니다.
- 날짜와 요일을 함께 보이게 하려면 문자열을 합치는 수식이 필요합니다.
- 데이터베이스 템플릿은 템플릿 목록에서 수정할 수 있습니다.
- `비어 있음`은 Notion 기본 항목이라 삭제가 어렵거나 불가능할 수 있습니다.

### 문제점

- 요일만 표시되고 날짜와 함께 표시되지 않았습니다.
- 템플릿 수정 위치가 직관적이지 않았습니다.
- `비어 있음` 템플릿을 일반 템플릿처럼 삭제하려고 했지만, 기본 제공 항목이라 삭제가 제한되었습니다.

### 해결 방안

- 날짜와 한글 요일을 함께 표시하는 수식을 적용합니다.
- 직접 만든 템플릿을 편집하고 기본값으로 설정합니다.
- `비어 있음`은 삭제하려고 하지 말고, 직접 만든 템플릿을 기본값으로 설정하여 대체합니다.
