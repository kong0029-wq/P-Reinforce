작성일: 2026-05-06

## 1. BOM Cost 비교 차트 가독성 개선

처음 검토한 차트는 여러 부품별 가격을 한 차트 안에서 비교하는 형태였습니다.

주요 항목은 다음과 같았습니다.

- Main Board
- RAM
- SSD
- OS
- USB HUB
- TOTAL

### 문제점

기존 차트에서는 데이터 라벨이 서로 겹치는 문제가 있었습니다.

특히 다음 항목에서 라벨 겹침이 발생했습니다.

- Main Board
- SSD
- OS
- 값이 비슷한 항목들
- TOTAL 근처의 큰 값들

### 해결 방향

차트 안의 숫자 라벨이 겹칠 때는 모든 라벨을 그대로 유지하기보다, 다음 방법을 적용하는 것이 좋다고 정리했습니다.

#### 1) 데이터 라벨 위치 조정

PowerPoint 또는 Excel 차트에서:

1. 숫자 라벨 클릭
2. 특정 숫자 라벨만 다시 클릭
3. 마우스 오른쪽 클릭
4. `데이터 레이블 서식`
5. 레이블 위치 조정

추천 위치:

| 시리즈 | 추천 라벨 위치 |
|---|---|
| 회색 시리즈 | 위쪽 또는 안쪽 끝 |
| 초록 시리즈 | 바깥쪽 끝 |
| 파랑 시리즈 | 위쪽으로 수동 이동 |

#### 2) 라벨 글자 크기 축소

추천 설정:

- 10pt → 8pt 정도
- 굵게 해제
- 검정 배경이면 연회색 또는 흰색 계열 사용
- 흰 배경이면 진회색 사용

#### 3) 모든 라벨을 다 표시하지 않기

값이 같거나 변화가 없는 항목은 라벨을 일부 생략하는 것이 좋다고 판단했습니다.

예:

- OS `$34 / $34 / $34`
- USB HUB `$35 / $35`
- SSD의 동일 또는 유사 값

이런 항목은 모든 숫자를 다 보여주기보다 하나만 표시하거나 생략하는 것이 더 깔끔합니다.

#### 4) TOTAL을 같은 차트에 넣지 않기

TOTAL 값은 다른 부품보다 값이 커서 축 범위를 크게 만들고, 작은 부품의 변화가 잘 보이지 않게 만듭니다.

그래서 TOTAL은 별도로 분리하는 것이 좋다고 정리했습니다.

## 2. TOTAL COST를 별도 표현하는 방식

TOTAL COST는 비교 대상이 적고 메시지가 명확하기 때문에, 일반 막대그래프보다 KPI 카드형 또는 별도 간단 차트로 표현하는 것이 좋다고 판단했습니다.

### TOTAL을 별도로 빼는 이유

기존 차트에서는 TOTAL 값이 커서 다음 문제가 발생했습니다.

- 축이 TOTAL 기준으로 잡힘
- RAM, SSD, OS 같은 부품 변화가 작아 보임
- 시선이 TOTAL에만 집중됨
- 부품별 원인 분석이 약해짐

따라서 다음 구조를 추천했습니다.

```text
상단: TOTAL COST KPI
하단: 부품별 원인 분석 차트
```

### 추천 구성

#### 상단 KPI

```text
TOTAL BOM COST
$385 → $509
▲32%
```

#### 하단 차트

- Main Board
- RAM
- SSD
- OS
- USB HUB

만 표시하고 TOTAL은 제거.

이렇게 하면 부품별 변화가 더 잘 보입니다.

## 3. KPI 카드형이란 무엇인가

KPI 카드형은 PowerPoint의 차트 삽입 메뉴에서 선택하는 차트가 아니라, 도형과 텍스트를 조합해서 숫자를 크게 강조하는 방식입니다.

즉:

| 항목 | 의미 |
|---|---|
| 세로 막대형 | 일반 차트 |
| KPI 카드형 | 도형 + 큰 숫자 + 설명 텍스트 |

### KPI 카드형 예시

```text
┌────────────────────┐
│   TOTAL BOM COST   │
│                    │
│   $385 → $509      │
│                    │
│      ▲32%          │
└────────────────────┘
```

### PowerPoint에서 만드는 방법

1. `삽입`
2. `도형`
3. `둥근 사각형`
4. 안에 텍스트 입력
5. 숫자와 증감률을 크게 표시

추천 글자 크기:

| 항목 | 추천 크기 |
|---|---|
| TOTAL BOM COST | 16~20pt |
| 금액 | 32~40pt |
| 증가율 | 20~24pt |

추천 색상:

| 요소 | 추천 |
|---|---|
| 배경 | 흰색 |
| 테두리 | 연회색 |
| 증가율 | 빨강 |
| 숫자 | 진회색 |

## 4. 3개 표의 TOTAL 가격 정리

검토한 표는 다음 3개였습니다.

1. 2025.07.31 기준 MIO-2364N
2. 2026.04.09 기준 MIO-2364N RAM 8G
3. 2026.04.09 기준 MIO-2364N RAM 16G

### TOTAL 값

| 구분 | TOTAL |
|---|---:|
| 2025.07.31 MIO-2364N | $272 |
| 2026.04.09 MIO-2364N RAM 8G | $888 |
| 2026.04.09 MIO-2364N RAM 16G | $1,011 |

### 기존 KPI 카드 문제점

처음 작성한 KPI 카드에는 다음처럼 숫자가 표시되어 있었습니다.

```text
TOTAL COST

$272
→ $888
→ 1,011

▲32%
```

이 표현은 몇 가지 문제가 있었습니다.

- 어떤 값이 어떤 모델인지 바로 알기 어려움
- `▲32%`가 실제 증가율과 맞지 않음
- `$272 → $888`은 약 `▲226%`
- `$272 → $1,011`은 약 `▲272%`

### 추천 표현

```text
TOTAL COST

2025.07      $272
2026.04 8G   $888
2026.04 16G  $1,011

8G  : ▲226%
16G : ▲272%
```

### 더 보기 좋은 카드형 예시

```text
┌────────────────────┐
│     TOTAL COST     │
│                    │
│ 2025.07      $272  │
│ 2026.04 8G   $888  │
│ 2026.04 16G $1,011 │
│                    │
│ 8G  : ▲226%        │
│ 16G : ▲272%        │
└────────────────────┘
```

### 핵심 메시지

이 표의 핵심 메시지는 다음과 같습니다.

```text
세대 변경 후 TOTAL COST가 크게 상승했고,
특히 RAM 가격 상승이 주요 원인이다.
```

## 5. RAM 가격 비교와 증가율 계산

RAM 가격 비교 차트에서는 다음 값들이 있었습니다.

| 구분 | 가격 |
|---|---:|
| 기존 DDR5 16G | $53 |
| 신규 DDR5 8G | $497 |
| 신규 DDR5 16G | $620 |

### 배수 기준 비교

| 비교 | 배수 |
|---|---:|
| $497 / $53 | 약 9.4배 |
| $620 / $53 | 약 11.7배 |

### 증가율 기준 비교

증가율 공식:

```text
(신규 가격 - 기존 가격) / 기존 가격 × 100
```

#### $53 → $497

계산:

```text
(497 - 53) / 53 × 100
= 444 / 53 × 100
≈ 837.7%
```

표기:

```text
▲838%
```

#### $53 → $620

계산:

```text
(620 - 53) / 53 × 100
= 567 / 53 × 100
≈ 1,069.8%
```

표기:

```text
▲1,070%
```

### 추천 차트 표기

```text
$497
▲838%
```

```text
$620
▲1,070%
```

또는 차트 중앙에 다음처럼 표시:

```text
8G  ▲838%
16G ▲1,070%
```

### 문구 추천

```text
RAM cost increased by up to 1,070% compared to 2025 baseline.
```

또는 짧게:

```text
Up to ▲1,070%
```

## 9. 간트차트 기본 생성 방법

PowerPoint 기본 차트에서 간트차트를 만들려면 `누적 가로 막대형`을 사용해야 합니다.

### 차트 선택 방법

1. `삽입`
2. `차트`
3. 왼쪽 메뉴에서 `가로 막대형`
4. 상단 아이콘 중 `누적 가로 막대형` 선택

주의:

- 첫 번째 `묶은 가로 막대형`이 아니라
- 두 번째 `누적 가로 막대형`을 선택해야 함

### 이유

간트차트는 다음 구조로 만들어집니다.

| 데이터 | 역할 |
|---|---|
| 시작 | 막대가 시작되는 위치를 만들기 위한 dummy 값 |
| 기간 | 실제 보이는 일정 막대 |

## 10. 간트차트 데이터 입력

입력한 데이터는 다음과 같았습니다.

| 제품 | 시작 | 기간 |
|---|---:|---:|
| i-Smart 30 | 1 | 3 |
| i-Smart 300 | 3 | 2 |
| i-Smart Care10 | 5 | 4 |

이 데이터 구조는 간트차트 생성 방식으로 맞는 입력 방식입니다.

### 의미

| 제품 | 시작 값 | 기간 값 | 의미 |
|---|---:|---:|---|
| i-Smart 30 | 1 | 3 | 1칸 뒤 시작, 3칸 지속 |
| i-Smart 300 | 3 | 2 | 3칸 뒤 시작, 2칸 지속 |
| i-Smart Care10 | 5 | 4 | 5칸 뒤 시작, 4칸 지속 |

### 핵심 작업

차트를 만들고 나서 `시작` 막대를 숨겨야 합니다.

방법:

1. 진한 파란색 `시작` 막대 클릭
2. 마우스 오른쪽 클릭
3. `데이터 계열 서식`
4. `채우기 없음`
5. `선 없음`

그러면 앞부분 dummy 막대가 사라지고 실제 기간 막대만 남습니다.

## 12. X축 숫자 변경 방법

PowerPoint 기본 차트에서 X축 숫자 `1, 2, 3, 4...`를 직접 Q1/Q2로 바꾸는 것은 번거롭습니다.

그래서 추천한 방법은 다음과 같습니다.

### 추천 방식

1. 기존 X축 숫자를 삭제
2. 아래에 텍스트박스로 직접 분기 표시

예:

```text
2025                     2026
Q1    Q2    Q3    Q4     Q1    Q2
```

실무 PPT에서는 축을 억지로 바꾸기보다 텍스트 박스를 직접 배치하는 방식이 훨씬 깔끔합니다.

## 13. 생산 수량 표시 위치

생산 수량은 간트차트에서 막대 끝 오른쪽에 넣는 것이 가장 좋다고 정리했습니다.

### 추천 형태

```text
i-Smart 30      ●──────────────  1,152 EA
i-Smart 300         ●──────────  424 EA
i-Smart Care10             ●───  20 EA
```

### 이유

- 막대는 일정 표현
- 오른쪽 숫자는 생산량 표현
- 역할이 분리되어 가독성이 좋아짐

### PowerPoint 입력 방법

1. `삽입`
2. `텍스트 상자`
3. 막대 끝 오른쪽에 수량 입력
4. 제품별로 수량 정렬

추천 스타일:

| 항목 | 추천 |
|---|---|
| 글자 크기 | 10~12pt |
| 색상 | 진회색 또는 검정 |
| 굵기 | Bold 가능 |
| 위치 | 막대 끝 오른쪽 |
| 단위 | EA 추가 추천 |

최종적으로 다음과 같이 표시했습니다.

| 제품 | 생산 수량 |
|---|---:|
| i-Smart 30 | 1,152 EA |
| i-Smart 300 | 424 EA |
| i-Smart Care10 | 20 EA |

## 14. 세로선 제거 방법

차트 안에 보이는 세로선은 `주 눈금선`입니다.

### 삭제 방법 1

1. 세로 흰색 선 하나 클릭
2. 모든 세로선 선택됨
3. Delete 누르기

### 삭제 방법 2

1. 세로선 클릭
2. 우클릭
3. `주 눈금선 서식`
4. `선 없음`

### 디자인 추천

- 세로선을 완전히 제거하거나
- 연회색으로 아주 연하게 유지
- 투명도 60~70% 정도 권장

## 15. 막대를 원형 노드 + 선 형태로 만드는 방식

처음에는 일반 간트 막대였지만, 더 프로다운 느낌을 위해 `원형 노드 + 선 형태`를 추천했습니다.

### 목표 형태

```text
●──────────────
```

의미:

| 요소 | 의미 |
|---|---|
| 원 | 적용 시작 시점 |
| 선 | 적용 기간 |
| 오른쪽 숫자 | 생산 수량 |

### 기존 차트를 유지하면서 바꾸는 방법

차트를 삭제하지 않고 다음 방식으로 개선했습니다.

#### 1) 막대 높이 얇게 만들기

1. 막대 클릭
2. `데이터 계열 서식`
3. `계열 옵션`
4. `간격 너비`를 크게 조정

추천:

```text
250% ~ 400%
```

이렇게 하면 막대가 얇아져 선처럼 보입니다.

#### 2) 부드러운 가장자리 적용

PowerPoint 차트 막대는 완전한 둥근 선을 자동 지원하지 않지만, 효과를 통해 비슷하게 만들 수 있습니다.

설정 위치:

1. 막대 선택
2. `데이터 계열 서식`
3. 효과 아이콘 선택
4. `부드러운 가장자리`
5. 크기 조정

추천 값:

```text
2pt ~ 5pt
```

가장 추천:

```text
3pt
```

#### 3) 시작점 원 추가

PowerPoint 차트는 시작 노드 원을 자동으로 만들 수 없기 때문에, 원은 도형으로 직접 추가해야 합니다.

방법:

1. `삽입`
2. `도형`
3. `타원`
4. Shift 누른 채 드래그해서 정원 생성
5. 막대 시작점에 배치

추천 크기:

```text
8~12pt
```

추천 색상:

| 요소 | 추천 |
|---|---|
| 라인 | 파랑 또는 연녹색 |
| 원 | 라인과 같은 색 또는 더 진한 색 |
| 흰 배경 | 진파랑 원 추천 |
| 검정 배경 | 흰색 또는 진녹색 원 추천 |

## 16. 타임라인 월/분기 표기 검토

중간 검토에서 `5월`이 `Q3`에 가까워 보이는 문제가 있었습니다.

### 분기 기준

| 분기 | 월 |
|---|---|
| Q1 | 1~3월 |
| Q2 | 4~6월 |
| Q3 | 7~9월 |
| Q4 | 10~12월 |

따라서:

```text
5월 = Q2
```

입니다.

만약 5월이 Q3 위치에 있으면 타임라인이 왜곡된 것입니다.

### 최종 방향

월 표기는 유지하기로 했습니다.

이유:

- 실제 적용 시점이 중요함
- 분기만 표시하면 정보가 다소 부족함
- 생산 일정 자료에서는 월 표기가 실무적으로 더 유용함

따라서 최종 차트에는 다음 월 표기를 유지했습니다.

| 제품 | 월 |
|---|---|
| i-Smart 30 | 1월 |
| i-Smart 300 | 5월 |
| i-Smart Care10 | 3월 |

다만 월 위치는 실제 분기 위치와 일치하도록 배치해야 합니다.

## 17. 최종 차트 디자인 검토

최종 차트 제목:

```text
완충 펌프 적용 분석기 생산 현황
```

구성:

- 왼쪽: 제품명
- 가운데: 원형 노드 + 라인
- 아래: 연도/분기 타임라인
- 오른쪽: 생산 수량

### 최종 표시 내용

| 제품 | 적용 시점 | 생산 수량 |
|---|---|---:|
| i-Smart 30 | 1월 | 1,152 EA |
| i-Smart 300 | 5월 | 424 EA |
| i-Smart Care10 | 3월 | 20 EA |

### 좋아진 점

최종 수정 후 좋아진 점은 다음과 같습니다.

#### 1) 생산 수량에 EA 추가

처음에는 숫자만 있었습니다.

```text
1,152
424
20
```

최종적으로는 단위를 붙였습니다.

```text
1,152 EA
424 EA
20 EA
```

이로 인해 숫자의 의미가 명확해졌습니다.

#### 2) 숫자 크기 축소

생산 수량 숫자가 너무 크면 일정 라인보다 숫자가 먼저 보입니다.

최종 수정 후 숫자 크기가 적절해졌고, 차트 전체 균형이 좋아졌습니다.

#### 3) 제목 크기 조정

초기에는 제목이 너무 커서 차트보다 제목이 먼저 보였습니다.

최종 수정에서는 제목 크기를 줄여 전체 균형이 좋아졌습니다.

#### 4) 하단 연도 바 개선

연도 바가 너무 크지 않게 조정되어, 본 차트가 더 잘 보이게 되었습니다.

#### 5) 원형 노드와 라인 적용

일반 막대형보다 훨씬 현대적인 로드맵 스타일이 되었습니다.

## 18. 최종 차트 추가 개선 포인트

최종본 기준으로도 더 프로다운 느낌을 위해 다음을 추천했습니다.

### 1) 월 텍스트 위치

월 텍스트는 유지하되, 원과 너무 붙어 보이지 않게 약간 아래로 내리는 것이 좋습니다.

추천:

```text
원 아래에서 5~10px 정도 여백
```

### 2) 월 텍스트 크기

월 텍스트가 너무 크면 시선이 분산됩니다.

추천:

```text
현재 대비 80~85% 정도
```

### 3) 세로 그리드선

세로 그리드선은 완전히 삭제하거나 아주 연하게 유지하는 것이 좋습니다.

추천:

```text
밝은 회색
투명도 60~70%
```

### 4) 수량 정렬

오른쪽 수량 텍스트는 모든 행에서 동일한 기준선으로 정렬해야 합니다.

추천:

- 라인 끝에서 일정 간격 유지
- 세로 중앙 정렬
- 오른쪽 숫자 컬럼처럼 정렬

### 5) 전체 여백

차트가 너무 꽉 차지 않도록 다음 여백을 유지합니다.

- 제목과 차트 사이 여백
- 제품명과 라인 사이 여백
- 라인과 수량 사이 여백
- 하단 타임라인과 라인 사이 여백

## 19. 최종 평가

최종 차트는 다음 요소를 모두 포함하게 되었습니다.

| 요소 | 반영 여부 |
|---|---|
| 제품명 | 반영 |
| 적용 월 | 반영 |
| 적용 기간/로드맵 | 반영 |
| 연도 구분 | 반영 |
| 분기 구분 | 반영 |
| 생산 수량 | 반영 |
| EA 단위 | 반영 |
| 원형 노드 | 반영 |
| 선형 로드맵 | 반영 |
| 제목 정리 | 반영 |

최종적으로 충분히 발표용, 고객 설명용, 내부 보고용으로 사용할 수 있는 수준이라고 평가했습니다.

점수 기준으로는 다음 수준입니다.

```text
초기 차트: 약 6~7 / 10
중간 수정본: 약 8~8.5 / 10
최종 수정본: 약 9.2 / 10
```

추가로 월 텍스트 위치, 수량 정렬, 그리드선 투명도까지 미세 조정하면 `9.5 / 10` 수준까지 올라갈 수 있습니다.

## 20. 오늘 정리한 핵심 원칙

오늘 작업에서 정리한 차트 디자인 원칙은 다음과 같습니다.

### 1) TOTAL은 부품별 차트에서 분리

TOTAL 값은 다른 부품보다 커서 축을 왜곡하므로, KPI 카드 또는 별도 차트로 분리하는 것이 좋습니다.

### 2) 가격 상승은 숫자보다 증가율과 배수를 함께 표시

예:

```text
$53 → $620
▲1,070%
11.7x
```

### 3) 모든 데이터 라벨을 다 넣지 않기

중복되거나 변화 없는 값은 생략해도 됩니다.

### 4) 차트에는 메시지만 남기기

차트는 모든 정보를 넣는 곳이 아니라, 핵심 메시지를 빠르게 전달하는 곳입니다.

### 5) 적용 일정은 막대그래프보다 로드맵 스타일이 더 적합

제품별 적용 시점과 생산 수량을 같이 보여줄 때는:

```text
원형 노드 + 라인 + 수량
```

형태가 가장 효과적입니다.

### 6) 생산 수량은 막대 끝 오른쪽에 배치

수량은 별도 열처럼 정렬하면 가독성이 좋아집니다.

### 7) 실무 PPT에서는 차트 + 도형 조합을 많이 사용

PowerPoint 기본 차트만으로 완성하려고 하지 않고, 도형/텍스트 박스를 추가해 마무리하는 것이 더 프로다운 결과를 만듭니다.

## 22. 최종 결론

오늘 작업의 핵심 결론은 다음과 같습니다.

```text
1. TOTAL COST는 별도 KPI로 분리하는 것이 좋다.
2. RAM 가격 상승은 증가율로 강하게 보여주는 것이 효과적이다.
3. 부품별 원가 차트에서는 중복 라벨을 줄여 가독성을 높여야 한다.
4. 생산 현황 표는 간트차트보다 원형 노드 + 라인형 로드맵이 더 프로답다.

6. PowerPoint에서는 기본 차트에 도형과 텍스트를 조합해야 완성도가 높아진다.
```

최종적으로 만든 방향은 발표자료, 내부 보고자료, 고객 설명자료로 모두 사용할 수 있는 수준입니다.
