# 📋 Tally-Notion-Solapi 자동화 구축 및 트러블슈팅 가이드

이 문서는 **Tally** 설문 응답을 **Make**를 통해 **Notion**에 저장하고 **Solapi** 알림톡을 발송하는 과정에서 발생한 주요 문제와 해결 절차를 기록합니다.

---

## 1. 노션 데이터베이스 연결 및 404/504 오류 해결

### 🔍 문제 원인
* **404 Error**: 메이크(Make)의 연결 권한(Connection) 설정 시 해당 노션 페이지가 선택되지 않았거나, 주소창의 '페이지 ID'를 '데이터베이스 ID'로 착각하여 발생.
* **504 Error**: 서버 간 일시적인 통신 지연으로 발생.
* **ID 불일치**: 노션의 '인라인 뷰' 주소와 '원본 데이터베이스' 주소가 서로 다름.

### ✅ 해결 절차 (ID 확인 및 권한 부여)
1.  **진짜 DB ID 추출**: 노션에서 해당 표의 제목을 클릭해 **[전체 페이지로 열기]**를 실행. 주소창의 `notion.so/` 뒤부터 `?v=` 앞까지의 **32자리 문자열**을 복사.
2.  **권한 재설정**: Make의 Notion 모듈에서 `Connection` -> `Add` 클릭 후, 노션 팝업창에서 **[페이지 선택]**을 눌러 해당 DB를 반드시 체크(V).
3.  **새로고침**: ID를 입력했는데 필드가 나타나지 않으면 `Data Source ID` 옆의 **[Refresh]** 버튼을 클릭.

## 2. 노션 400 Runtime Error (데이터 형식 불일치)

### 🔍 문제 원인
* **Array vs String**: Tally의 객관식 답변은 시스템적으로 **배열(Array, `["값"]`)** 형태로 전송됨.
* **속성 충돌**: 노션의 **'선택(Select)'** 속성은 단일 문자열(String)만 받을 수 있는데, 대괄호가 포함된 배열이 들어오면서 유효성 검사 실패.

### ✅ 해결 절차 (단일 선택 속성 유지 시)
1.  Make의 노션 설정창에서 에러가 발생한 필드의 기존 블록을 삭제.
2.  상단 함수 탭에서 **배열 함수(Array Functions)** 아이콘을 클릭.
3.  **`first()`** 함수를 선택하여 입력창에 생성.
4.  괄호 `( )` 안에 Tally의 해당 질문 데이터 블록을 삽입.
    * *형태: `first( 8. 질문 내용 [ ] )`*
5.  이렇게 하면 `["답변"]`에서 대괄호를 제거한 `"답변"`만 노션으로 전송됨.

## 3. 실시간 자동 실행 설정 (Scheduling)

### 🔍 문제 원인
* `Run once`를 누를 때만 작동하고, 실제 설문 제출 시에는 자동 실행되지 않음.

### ✅ 해결 절차
1.  Make 시나리오 하단의 **Scheduling 토글 스위치**를 **ON**으로 변경.
2.  `Run scenario` 설정이 **Immediately** (데이터 도착 시 즉시)인지 확인.
3.  **Maximum runs to start per minute**: 1분당 최대 실행 횟수를 설정 (일반적으로 **10~20** 권장).
4.  왼쪽 하단의 **디스크 모양 아이콘(Save)**을 눌러 최종 저장.

## 4. 솔라피(Solapi) 계정 및 연동 관리

### 🔍 문제 원인
* 기존의 다른 Make 계정에서 솔라피 API Key를 사용하여 중복 실행되거나, 접속이 불가능한 계정의 연동을 끊어야 하는 경우.

### ✅ 해결 절차 (솔라피 관리자 페이지)
1.  [솔라피 홈페이지] 접속 -> [관리자/개발자 센터] -> **[API Key 관리]** 이동.
2.  **연동 해제**: 사용 중인 API Key 목록에서 삭제하려는 키를 **[삭제]** 처리. (해당 키를 가진 모든 외부 서비스의 접근이 즉시 차단됨)
3.  **보안 유지**: 새로운 API Key를 발급받아 현재 사용 중인 Make의 Connection에만 업데이트하여 깨끗한 상태 유지.
